**AI 时代，会阅读理解你就会编程**

上大学的时候，我们实验室有个师妹， 一开始也觉得编程很难，不知道怎么开始做实验（我们并非计算机专业本科，只上过几门基础计算机课程）。

因为我们是一个项目组的，我就对着代码一行一行给她讲了几次，然后教她怎么把实验跑通，怎么看实验结果（比如有哪些指标）。之后，她就自己去尝试了。

有一天我们再聊起编程，她说： “**写代码好像写文章啊！**”。

这是我第一次听人把「**编程和写作**」放在一起比较，所以多年后的今天，我依然印象深刻。

而且，我越来越认可。因为**计算机语言，首先是写给人看的**。

大多数时候，写软件靠的是团队协作：写出来的代码别人能看懂， 协作的效率就会更高。即便独立开发（只有自己一个人写），我们也是在团队协作 —— 和未来的自己协作。

大多数有用的软件，都是需要持续维护。我们常常需要在多个项目之间切换，或者在不同工作之间切换。如果代码写完，过几天就看不懂了，首先难受的是自己。

从这个角度看，写代码很像写笔记/写教程，其实是写给未来的自己的小抄。

今天，在 AI 的帮助下，这件事更简单了，因为 AI 可以生成可用的代码了。

如果不是很复杂的项目，你甚至可以把代码当做黑盒，只负责管控软件代码的输入输出。

在 Claude Code 这种灵活的工具加持下，你要维护的其实是一堆 Markdown 文件而已。所以，只要你会阅读理解，你就会编程。如果我们常说的「写作」是「用自然语言写作」，「写代码」其实就是「用编程语言写作」。

**当然，能看懂代码更好，长期来看也很重要**。因为 **AI 其实是能力放大器**，你对一个领域越擅长，AI 就越能放大你的能力。你观察**身边**那些 AI 用得好的人，大多就是非常擅长一个领域的人，然后通过 AI 放大了他的能力。

在编程领域也一样，能把 AI 用好的，也是那些原本就能写好代码、会用各种工具提升代码质量的人。

**的确，编程是有门槛的，但真的是一层窗户纸**。捅破之后，就是新的世界。

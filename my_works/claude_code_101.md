# Claude Code 101：不会写代码，也能用 AI 做事

> 把 AI 从聊天框里放出来，让它直接在你电脑上干活

先说一件事：你正在看的这篇文章，是用 Claude Code 的写作系统生产的。从选题、调研、写初稿到审稿，每一步都是在终端里用自然语言指挥 AI 完成的。我不是程序员出身，文章里没有一行代码是手写的。

如果你的反应是"等等，这怎么做到的？" —— 这就是我想聊的事。

## 聊天框是 AI 的笼子

你大概率用过 ChatGPT 或者网页版 Claude。输入一个问题，AI 给你一段回答，你再问，它再答。用久了，你可能觉得 AI 就是这样 —— 一个聊天搭子，能写文案、能翻译、能回答问题，但也就这样了。

问题不在 AI 不够强，而在聊天框把它关住了。

网页版 AI 的活动范围就是那个对话框。它能接收你输入的文字，输出一段文字，但它碰不到你电脑上的文件，不能执行任何操作，也连不上你正在用的其他工具。每次跟它聊，就像去酒店开房 —— 每次都要重新介绍自己，它也不知道你房间里长什么样。

Claude Code 不一样。它运行在你的电脑上，能直接读写文件、执行命令、操作你的项目。用 Dan Shipper 的比喻：网页版像住酒店，Claude Code 像有 AI 的自己的公寓 —— 它知道你的东西在哪，能帮你整理，能帮你干活。

这里有个细节要说清楚：说"运行在你电脑上"，指的是它的操作在本地执行 —— 读你的文件、改你的文件、跑你的程序。但 AI 的"思考"过程仍然在云端完成，所以需要联网。

通过一种叫 MCP（Model Context Protocol）的开放标准，Claude Code 还能连接外部数据源和工具 —— Google Drive、Jira、Slack 等等。这意味着它不只是一个操作本地文件的工具，而是一个能伸手到外部世界的 AI 助手。

## 你需要准备什么

开始之前，确认这几样东西：

- **一台电脑** —— macOS、Windows 或 Linux 都行
- **Claude 付费账号** —— Pro（$20/月）起步就能用 Claude Code。免费账号不行
- **网络** —— Claude Code 需要联网（AI 思考在云端）

不需要提前装任何编程工具，不需要会用终端，不需要懂代码。

## 安装：一行命令搞定

Claude Code 提供了桌面版和终端版两种入口。如果你对终端有心理障碍，**先用桌面版**，界面跟普通软件没什么区别。

**桌面版（推荐新手）**

去 [claude.ai](https://claude.ai) 下载桌面 App，安装后登录你的 Claude 账号就能用。

**终端版**

打开终端（macOS 上叫"终端"或"Terminal"），粘贴这行命令，回车：

```bash
curl -fsSL https://claude.ai/install.sh | bash
```

Windows 用户用 PowerShell：

```powershell
irm https://claude.ai/install.ps1 | iex
```

装完后，在终端输入 `claude`，回车。第一次会让你登录 Claude 账号，跟着提示走就行。

**怎么确认装好了？** 输入 `claude --version`，能看到版本号就说明成功了。

## 第一个任务：让 AI 帮你整理文件

装好了，然后呢？我建议从一个你马上能感受到价值的任务开始。

比如，你的桌面或者下载文件夹里有一堆乱七八糟的文件 —— 截图、PDF、文档混在一起。打开 Claude Code，告诉它：

```
帮我看看 ~/Downloads 文件夹里有什么文件，按类型分类整理到不同的子文件夹里
```

Claude 会先列出文件清单给你看，然后提出一个整理方案。你确认后，它就动手了 —— 创建文件夹、移动文件，全部自动完成。

整个过程你说的是中文，Claude 执行的是操作。你不需要知道 `mv` 命令怎么写，不需要知道文件路径的语法。

再来几个例子，感受一下"用自然语言操作电脑"是什么体验：

- "把这 200 张照片按拍摄日期分到不同文件夹"
- "从这份 PDF 里提取所有表格数据到 Excel"
- "把这个文件夹里所有 .docx 文件转成 PDF"

你能用语言清楚描述想要什么结果，但不知道具体怎么操作 —— 这类任务就是 Claude Code 的甜蜜区。

## 不会写代码，能用来干什么

衡量能不能用 Claude Code 的标准，不是"会不会写代码"，而是"能不能描述你想要什么结果"。

用我自己的写作系统来说明。这套系统有一组预先定义好的 Skills（可以理解为"工作流程配方"），每个用一个斜杠命令触发：

- `/brief` —— 我描述一个选题，AI 提出几个角度供我选，然后通过几轮对话明确任务边界，最终生成一张任务卡
- `/research` —— 根据任务卡联网搜索资料，交叉验证事实，生成一份标注了可信度等级的证据地图
- `/outline` —— 把任务卡和证据地图组织成文章提纲
- `/draft` —— 按提纲逐段写出初稿
- `/review` —— 独立审稿，检查事实准确性、表述强度是否越界

全程用中文自然语言交流。我要做的，就是在每一步给出判断：这个角度好不好？这个结构对不对？这段表述准不准？AI 负责执行，人负责决策。

这些 Skills 本质上就是预先写好的工作流程，一个命令触发一套动作。你不需要知道背后怎么实现的，就像你不需要知道洗衣机内部怎么转，只需要选好模式按下开始。

有个边界要说清楚：**搭建这套写作系统本身需要技术投入，但使用它不需要。** 一旦系统搭好，操作者只需要会用自然语言描述需求。

写作只是一个例子。已有不少非技术背景的用户在用 Claude Code 做其他工作：费用追踪、文件整理、数据分析、批量内容处理。Helen Lee Kupp 的经历很有代表性 —— 她不是开发者，起初对终端充满恐惧，但用了 Claude Code 之后发现，"这个工具不是关于编码的"。

## 怎么找到你的任务

知道 Claude Code 能干什么，跟找到自己用它干什么，中间还隔着一步。这一步需要足够的使用密度来跨过。

这不是玄学。有一个叫"30 倍法则"的思维框架可以帮你理解这笔投入：如果你每天有一个重复性任务花 10 分钟，花 300 分钟（大约 5 小时）去学习怎么自动化它，30 个工作日后就回本了，剩下的 220 个工作日全是净赚的时间。这个类比不是精确计算，但逻辑方向是对的：前期投入的时间，后面会以倍数还回来。

我的经验是从三个维度去找：

**1. 重复 3 次法则（频率）**

回顾过去一周，有哪些电脑操作你重复做了 3 次以上？批量改文件名、从表格里提取特定数据、把一种格式转成另一种……这些重复劳动都值得交给 Claude Code 试试。

**2. 全天浸泡日（广度）**

找一天不那么忙的日子，把当天所有任务都尝试用 Claude Code 来做。不用期待每件事都成功，目的是发现：哪些任务出乎意料地顺，哪些完全不行。一天下来你就有了一张"适合/不适合"的清单。

**3. "会说不会做"（能力差）**

你能用语言清楚描述想要什么结果，但不知道具体怎么操作 —— 这类任务是 Claude Code 的甜蜜区。"把这 200 张照片按拍摄日期分文件夹"、"从这份 PDF 里提取所有表格" —— 你知道要什么，Claude Code 知道怎么做。

这三个方法来自我个人的使用经验，不是什么"经过验证的方法论"。但它们指向一个核心：先用起来，在使用中发现可能性。

## 为什么要配合 git —— 你的后悔药

你可能注意到前面提到的写作系统里，每一步完成后都会自动"存档"。这个存档系统就是 git。

git 不是程序员专用的神秘工具。你可以把它理解为一个**时间机器式的存档系统** —— 每次存档记录了项目在那个时刻的完整状态，你随时可以回到任何一个存档点。

为什么这对 Claude Code 用户特别重要？因为 AI 不是永远正确的。Claude 可能把一个文件改坏了，可能删了不该删的东西，可能某个方向走偏了。有了 git，这些都不是问题 —— "帮我回到上一步"，一句话搞定。

git 还有一个强大的功能：**分支**。你可以让 Claude 在一个分支上尝试方案 A，在另一个分支上尝试方案 B，两个方向互不干扰。试完了，你来决定选哪个。

你不需要先学会 git 命令才能用。Claude Code 帮你执行 git 操作 —— 你用自然语言说"帮我保存当前状态"或者"回到上一步"，它会帮你跑对应的命令。先用起来，在使用中自然会慢慢熟悉。

**安装 git：** macOS 通常自带。Windows 用户去 [git-scm.com](https://git-scm.com) 下载安装，一路默认就行。装完后在终端输入 `git --version`，有版本号就成功了。

## 从这里开始

你现在从 AI 拿到的，很可能只是它能给你的一小部分。扩大这个空间的门槛，比你想象的低得多。

不需要先学会终端，不需要先学会 git，不需要先学会任何你觉得"应该先学会"的东西。找一个任务，用自然语言告诉 Claude 你想要什么，然后看看会发生什么。

# Everything Claude Code：给你的 AI 编程助手装上「操作系统」

> 一个开源项目，把 Claude Code 从一个聪明的助手，变成一个有记忆、有技能、有安全意识的工程团队。

---

用 Claude Code 写了大半年代码，我一直有个困扰：**每次开新对话，它就像失忆了一样。**

上次聊好的代码规范？忘了。项目的架构决策？不记得。我偏好的测试风格？得重新教。

这感觉就像你招了一个天才实习生，能力很强，但每天早上到公司都不记得昨天做了什么。你得花半小时重新交代背景，才能开始干活。

后来我在 GitHub 上发现了 [Everything Claude Code](https://github.com/affaan-m/everything-claude-code)（简称 ECC），才意识到：**问题不在 Claude 不够聪明，而在于我没给它搭好工作环境。**

## 它到底是什么？

ECC 不是一个独立的工具，而是一套给 Claude Code 用的「配置增强包」——13 个专业 Agent、56 个 Skill、32 个命令，外加安全扫描、持续学习、跨平台 Hook 等等。

打个比方：Claude Code 出厂是一台裸机，ECC 给它装上了操作系统。

装完之后，你可以：

- 让专门的 Agent 做代码审查，而不是让一个通用 AI 什么都干
- 用 `/tdd` 命令强制测试驱动开发，先写测试再写代码
- 用 `/security-scan` 扫描你的 Claude 配置文件有没有安全漏洞
- 让系统自动从你的工作习惯中提取模式，下次自动应用

## 两个让我印象深刻的设计

**1. 专业分工的 Agent 体系**

ECC 把「一个 AI 干所有事」拆成了 13 个专业 Agent：Planner 负责规划、Architect 负责架构、Code Reviewer 负责审查、Security Reviewer 负责安全……

这和软件团队的分工逻辑一样。你不会让前端工程师去审数据库安全，AI 也不应该用同一个 prompt 处理所有任务。每个 Agent 有自己的工具集和关注点，做自己最擅长的事。

**2. 持续学习系统**

这是我觉得最有意思的部分。ECC 有一套叫「Instinct」的机制——它会观察你的编码习惯，提取出可复用的模式，存成「本能」。用得越多，置信度越高，最终会自动进化成 Skill 或 Command。

换句话说，你用得越久，它越懂你。这才是 AI 助手该有的样子。

## 安装很简单

最快的方式，一行命令：

```
/install everything-claude-code
```

也可以手动 clone 仓库，跑 `./install.sh [language]` 按语言安装规则。

装完之后，你的 Claude Code 就多了一堆新技能。原来的功能不受影响，纯粹是增量。

## 我的看法

ECC 代表了一个趋势：**AI 工具的价值，越来越取决于它的配置和上下文，而不仅仅是模型本身。**

同样是 Claude Opus，裸跑和装了 ECC 的体验差距很大。就像同一个人，给他一套趁手的工具和一个好的工作流程，产出完全不同。

当然，56 个 Skill 你不一定全用得上。我建议从自己最痛的点入手——如果你老是忘了写测试，先用 `/tdd`；如果你担心安全问题，先跑 `/security-scan`。别贪多，挑几个用熟了再加。

AI 编程的下一步，可能不是等一个更强的模型，而是把现有模型的潜力真正释放出来。ECC 做的就是这件事。

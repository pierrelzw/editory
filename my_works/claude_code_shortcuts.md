# Claude Code 的这些快捷键，我用了三个月才发现

> 别光顾着打字，手边还有不少趁手的操控键。

---

一直以来我用 Claude Code 的方式很单一：打开终端，输入问题，等回答，再输入。就这样。

屏幕底部偶尔冒出一行小字 —— "Press Ctrl+B to background this task" —— 我每次都看到了，但从来没按过。

把任务放到背景里运行？但我正在等结果呢，放背景运行干啥？

直到有一天，我发现 Claude  Code 跑得有点慢，而且，我还有补充的信息想告诉他。

等了快两分钟，它还在搜索信息、翻文件。我实在坐不住了，随手按了一下 Ctrl+B。

终端底部出现了 "1 agent"，然后…… 我发现我可以继续打字了。

那个搜索任务还在后台跑着，但我已经在问下一个问题了。等后台任务跑完，结果自动在会话里弹出来。

就这么简单的一个操作，我居然无视了三个月。

后来我开始翻 Claude Code 的快捷键列表，发现这种 "早知道就好了的快捷键” 还不止一个。

## 1. Ctrl+B — 放到后台，继续干别的

这是改变我使用习惯的第一个快捷键。

Claude 在跑耗时任务的时候 —— 搜索大量文件、执行测试、等 API 返回 —— 按 Ctrl+B 把它扔到后台。终端左下角会显示 "1 agent"，表示有一个后台任务在跑。你可以继续输入新问题，两边互不干扰。

后台任务跑完后结果会自动出现在对话里。

**注意**：如果你用 tmux，Ctrl+B 要按**两次**。因为 tmux 把 Ctrl+B 当作自己的前缀键，第一下会被它吃掉。

## 2. Ctrl+T — 看看后台都在跑什么

放了几个任务到后台，记不清哪个跑完了哪个还在转？Ctrl+T 打开任务列表，能看到所有任务的状态：进行中、已完成。

和 Ctrl+B 是一对搭档。

## 3. Ctrl+F — 后台任务跑飞了，杀掉

有时候后台任务卡住了，或者你发现方向不对不想要了。按 Ctrl+F，3 秒内再按一次确认，所有后台任务直接终止。

相当于后台任务的紧急制动。

## 4. Escape — 打断 Claude 的回答

Claude 开始回答，但方向明显不对 —— 你问的是怎么改配置，它在给你重写整个模块。

按 Escape，立刻停下来。然后你可以补充说明、换个问法、或者直接给新指令。不用等它说完再纠正。

这个快捷键我之前一直以为只能取消输入。其实在 Claude 输出的过程中按，也能打断它。

## 5. Esc + Esc — 时光倒流

这个是让我最惊喜的。

连按两下 Esc，会弹出一个交互式菜单，列出对话中每一轮的 checkpoint。选中某个节点后，有三个选项：

- **Restore code and conversation** — 代码和对话都回到那个时间点
- **Restore code** — 只回退代码，对话记录保留（Claude 还记得上下文）
- **Restore conversation** — 只回退对话，代码不动

还有一个 **Summarize from here** —— 不回退，而是把选中点之后的对话压缩成摘要，释放 context 窗口空间。

**原理**：Claude Code 在每次你发 prompt 之前会自动快照文件状态，这套 checkpoint 系统独立于 git。回退后，你之前输入的 prompt 会自动填回输入框，方便改了重发。

**但有个重要限制**：checkpoint 只追踪 Claude 通过自己的编辑工具（Read/Write/Edit）改的文件。如果它通过 bash 跑了 `rm`、`mv` 这类命令，那些改动是回退不了的。所以大改之前，还是先 git commit 比较稳。

## 6. Option+P — 切模型

简单问题用 Opus 有点浪费。按 Option+P（macOS）弹出模型选择器，切到 Sonnet 或 Haiku 处理轻量任务，复杂的再切回来。

不用退出对话，不用改配置文件，随时切。

（Windows/Linux 上是 Alt+P）

## 7. Option+T — 开关深度思考

有些问题需要 Claude 想深一点 —— 架构设计、复杂 debug、多文件重构。按 Option+T 开启 extended thinking，Claude 会先在内部推理一轮再回答。

简单问题关掉就好，省 token 也省时间。

首次使用需要先跑一下 `/terminal-setup` 启用这个快捷键。

（Windows/Linux 上是 Alt+T）

## 8. Shift+Tab — 切换权限模式

Claude 每次执行命令都要你点确认，有时候很烦 —— 特别是你信任它、只想让它全速跑完的时候。

Shift+Tab 在三种权限模式之间切换：

- **正常模式** — 每次执行都问你
- **Auto-Accept** — 自动批准所有操作
- **Plan 模式** — 只规划不执行，适合你想先看方案再动手的场景

跑测试的时候切到 Auto-Accept，改核心代码时切回正常模式。一个快捷键搞定。

## 9. `!` 前缀 — 不用退出 Claude 就能跑命令

想看一下 git status？想跑个 ls 确认文件在不在？

不用退出 Claude Code，也不用开新终端。直接输入 `! git status`，命令执行结果会直接出现在对话里，Claude 也能看到这个输出。

这样你就可以一边和 Claude 讨论，一边验证它说的对不对。

---

在此之前，我一直觉得 Claude Code 就是个"聊天框" —— 我打字，它回答。

但翻完这些快捷键才意识到，它更像是一个需要操控的协作工具。打断、放后台、回退、切模型……这些操作本质上是在调整**我和 AI 协同的节奏**。

等它的时候可以不干等。方向不对可以立刻喊停。改坏了可以倒带重来。简单问题不用杀鸡用牛刀。

把这些快捷键用起来以后，最大的变化不是效率提高了多少，而是一种感觉 —— 我不再是在"等 AI 干活"，而是在"指挥 AI 干活"。

# Claude Code Tab 太多了怎么办

> 关掉就好。真的。

---

最近每天打开电脑就是和 AI 协同工作，映入眼帘的就是 Claude Code 的 10 多个 Tab。

有的是上周调 bug 开的，有的是某天晚上突发灵感开的讨论，有的……我已经完全不记得为什么要开了。

我试着读了两个。读到一半，脑子里冒出一个念头："这是我现在最重要的事么？" 似乎不是。于是又放下了。

但我又不敢关。因为我隐约觉得这些是"未处理的内容" —— 我当时应该是遇到问题了才发起的聊天，万一关了以后又要重新问呢？我的确有过好几次开新 Tab 问了相同问题的经历。

于是，我就这么拖着。Tab 越来越多，我看着它们也越来越焦虑，但就是不动手。

我用的是 iTerm2，我尝试通过配置  profile 解决我的焦虑。但 profile 只是允许我通过 Tab 颜色区分不同项目，没有解决 “打开 Claude Code 的 Tab 太多” 的问题。

## 这个焦虑到底是什么

今天，我和 AI 仔细聊了很久，它告诉我：焦虑的来源不是"Tab 太多"，而&#x662F;**"未做决定的太多"**。

每个没关的 Tab，本质上是一个我对自己的微小承诺："我会回来处理的。" 这些承诺堆在那里，就像桌上堆了一叠没拆的快递 —— 我不知道里面是什么，但我知道"还没处理"，这件事本身就在消耗我。

David Allen 在《Getting Things Done》里说过：你的大脑不擅长存储承诺，但它会持续追踪它们。

每次我扫过 Tab 栏，大脑都在做一次微型决策 —— "要不要处理？" 决定"不处理"本身就消耗意志力。

**我不是 Tab 管理能力不足，我是在为 20 个未做的决定支付认知租金。**

## 我为什么不敢关

不敢关的理由通常是：”万一这里有一些有用的关键信息呢？"

这个担心听起来合理，但经不起推敲。

我问了自己一个问题：**上一次主动翻旧 Tab 找信息，是什么时候？**

我回忆了一下，答案是……想不起来。需要的时候，我都是直接开新 Tab 重新问。旧 Tab 就在那里，但我从来不去翻。

这说明 Tab 作为"外部记忆"是失败的。它不是一个我会去查阅的档案，它是一个只会越积越多但从不翻阅的旧报纸堆。

而且我后来意识到，反复问同一个问题，可能恰恰说明旧答案不值得保留。每次问的时候，上下文不一样了，我对问题的理解也不一样了，项目阶段可能也不一样了。

我需要的，是一个**基于当下状态的新回答**，不是三周前那个旧对话。

## 关掉的真实成本是多少

几乎为零。

Claude Code 的所有聊天记录都保存在本地 `~/.claude/projects/` 下的 JSONL 文件里。关掉 Tab 不是删除对话，只是关闭了窗口。数据还在，想找随时能找。

装了 [cccmemory](https://github.com/mako-makok/cccmemory-mcp)、**claude-mem** 这类工具的话，还能用自然语言搜索历史会话 —— "我上周调那个 ESLint 配置是怎么解决的？" ，有的时候比翻 Tab 还快。

但说实话，**大概率用不到。** 因为在 AI 时代，"重新获取"信息的成本趋近于零。忘了一个命令行参数？重新问，30 秒搞定。忘了上次的 debug 思路？重新描述问题，Claude 会基于当前代码状态给一个可能更好的方案。

旧世界的直觉是：信息是稀缺资源，获取了就要保存。

新世界的现实是：**信息是可再生资源，保存的投资回报率正在坍缩。**

## 直接关，然后观察

我的建议很简单：

**现在就把所有超过一周的 Tab 关掉。** 不读，不整理，直接关。

然后正常工作两周。如果某个被关掉的对话确实需要了 —— 重新问 AI，注意计时，看花了多久。

我发现了两件事：

- 绝大多数我**根本没想起来过**
- 少数需要的，重新问 30 秒就搞定了

做三四次之后，关 Tab 时就不会有那种沉甸甸的感觉了。

Nassim Taleb 说，“损失厌恶” 是进化赋予的默认设置 —— **在资源稀缺的环境里，"不确定有没有用就先留着"是最优策略**。但这个默认设置是为物质世界优化的。在 AI 时代，它刚刚过时。

不需要靠"勇气"克服它，只需要**反复体验"关了也没事"**，直觉就会自动校准。

## 如果还是不放心

养成一个小习惯：**对话结束时，花 10 秒写一句话。**

不是写笔记，不是做总结，就一句话 —— 这次对话解决了什么、或者没解决什么。写在项目的 TODO 里，写在备忘录里，发给自己一条消息，都行。

这一句话的意义不在于"保存信息"，而在于**完成那个承诺**。大脑不再追踪"那个 Tab 里有个未处理的事"，因为我已经对它做出了决定。

Allen 管这叫 "Clarify" —— 不是要读完它，而是用 10 秒决定它的命运。10 秒决定比 10 分钟阅读有效得多。

---

其实写这篇文章的过程本身就是一个例子。

这个话题来自一场和 AI 的圆桌讨论，我用 Claude Code 找来了前面提到的四个虚拟嘉宾吵了六轮，从信息管理吵到进化心理学。讨论很精彩。

但你问我现在还记得多少细节？大概 20%。我记住的是什么？是一个感觉 —— **"关掉没事的。"**

对话的价值在发生的那一刻就已经交付了。这篇文章也一样 —— 读完之后，关掉这个 Tab 吧。

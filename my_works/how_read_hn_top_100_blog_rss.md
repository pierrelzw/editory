Karpathy 推荐的 2025 最受欢迎的 92 个博客，如何快速订阅和每天阅读？



两天前，我打开 X，刷了好几屏，都在转发一个人把 Boris Cherny 总结的 13 的使用的 Claude Code 的方法，总结成了一个 CLAUDE.md。



我不知道这些转发的人是不是都试过了，但是我看不下去了，只好选择关闭。



AI 相关的信息流，想海浪一样，一波又一波。有的时候想看，但是稍不注意就被淹没，但是不看，又不自觉地产生 FOMO（害怕错过）的情绪。



于是，我问 AI：“在哪里可以最方便地看到 AI工具，尤其是 AI Coding 工具的更新进展、最佳时间案例？我想每天都花点时间看一下，有什么方案？”



我甚至想和 Claude Code 说：帮我开发一个工具。



不过，我忍住了。因为我意识到，这件事我似乎干过很多次。



我心想：“既然这么多次我都没有干，说明看这么多的信息，可能不是我真正的需求”。



紧接着，一系列的问题在我脑子里冒出来：

- 我自己写个工具，真的能挖掘到所谓优质信息么？

- 我平时没有刷这些信息的习惯，我真的知道 AI 领域的优质信息在哪里么？

- 我只看 AI 领域的信息么？……



当生产成本降低，生产变得容易，我们的注意力就会更加稀缺。—— 忘了谁说的了



我想起 Karpathy 在 X 上分享了 “2025 年 Hacker News 上最受欢迎的 92 个博客”，这似乎是一个还不错的信息源。



我记得在他分享前，就在 HN 上看过一个「2025 Top100 Blogs 」的帖子。但是，因为我之前没用过 RSS，也不怎么看博客，所以我就也没有认真研究。



今天想到这件事，我就问了一下 AI。。



它很快给我推荐了一些 RSS 阅读器，更重要的是，提醒了我其实Karpathy 分享的就是一个 RSS OPML 文件链接。在浏览器打开以后，点击右上下载 Zip 文件解压即可。这个文件本身就聚合了这 100 个博客的信息，直接导入 RSS 阅读器就好了。



我选了Inoreader（https://www.inoreader.com/）



注册登录以后，点击 Add》上传 OPML 文件，就可以 Feeds 里看到这些博客的近期文章了。



AI 还推荐把这些博客进行分类什么的。我心想：还没开始看呢，怎么知道怎么分？要不 “让 Claude Code 帮忙分？”



“别别别”，我摁住了这个冲动，暂时没必要，先看起来再说。

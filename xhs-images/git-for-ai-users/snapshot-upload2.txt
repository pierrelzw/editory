这是用 Claude Code 做项目的新手朋友，很容易踩的坑，还有下面这些类似的：

1️⃣ Claude Code 生成了 v1/v2/v3 的文件， 一周后分不清哪个是哪个了

2️⃣ CLAUDE.md 改着改着，Claude 行为变怪了，但不知道哪一次改坏的

3️⃣ 多人协作，几个人同时更改了一些文件，合并改动简直是灾难

4️⃣ 用 Claude Code 迭代一个月的写作风格，想知道是怎么演进的，但没留过快照

5️⃣ Claude 改了配置我没注意就同意了，几天后才发现有问题，但是不知道它到底改了什么




它们的共同点：没有「存档」。




好消息是，在软件工程领域早就有解决方案了 —— Git 就是你的存档键




其实这个工具很简单，不是程序员才会用的工具。下面是最基础的 6 个命令：

git init → 开启存档（只需要跑一次）

git add + commit → 存一个档（改完就存，养成习惯）

git log → 看看存了哪些档（像翻相册）

git diff → 对比改了什么（Claude 改了啥一目了然）

git checkout → 回到之前的版本（后悔药）

git branch →  开一条新路线试试（不影响主线）




你甚至可以直接让 Claude Code 执行它们。




💡 在 CLAUDE.md 加一条规则，Claude 就能自动帮你存档👇

「项目使用 git 管理，每次完成修改后自动 commit。」




最近，Claude Code 内置了 git worktree，比 git branch 更好用，感兴趣的可以找 Claude 了解。




在使用 Claude Code 初期你踩过哪个坑？评论区见👇

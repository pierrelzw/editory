# AI Content Review Checklist

Use this checklist when reviewing an article before publishing. Check each item and provide feedback.

## Formatting

- [ ] **Heading hierarchy** — Headings follow a logical order (H1 > H2 > H3), no skipped levels
- [ ] **No broken links** — All links are valid and properly formatted
- [ ] **No broken images** — All image references point to accessible files/URLs
- [ ] **Consistent formatting** — Bold, italic, code formatting used consistently
- [ ] **Code blocks** — Code blocks have language identifiers for syntax highlighting
- [ ] **Lists** — Lists are properly formatted (no orphaned items, consistent style)
- [ ] **No trailing whitespace** — Clean line endings

## Content Quality

- [ ] **Compelling title** — Title is clear, engaging, and accurately represents the content
- [ ] **Has summary/abstract** — Article includes a brief summary (for platforms that need it)
- [ ] **Clear structure** — Article has a logical flow with clear sections
- [ ] **Conclusion** — Article has a proper ending or call to action
- [ ] **Spelling & grammar** — No obvious typos or grammatical errors
- [ ] **Readability** — Sentences are not overly long or complex

## Metadata

- [ ] **Tags are relevant** — Tags accurately categorize the content
- [ ] **Sufficient tags** — At least 2-3 tags provided
- [ ] **Cover image** — Article has a cover image (or one should be generated)
- [ ] **Author info** — Author name is set (from frontmatter or config)

## Platform Readiness

- [ ] **Character limits** — Content can be adapted to fit platform limits (Twitter 280 chars, WeChat summary 120 chars)
- [ ] **Image requirements** — Images meet platform requirements (WeChat cover 900x383, Xiaohongshu vertical preferred)
- [ ] **Content suitability** — Content is appropriate for target platforms
